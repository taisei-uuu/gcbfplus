# 非等方スケーリング（面積保存）実装ガイド

## 1. 概要：等方スケーリングの問題と解決策

### 1.1 等方スケーリングの問題

従来の等方スケーリングでは、全ての方向に同じ係数でスケーリングするため、エージェント間の距離が縮小されます：

$$\|p_i^{d,scaled} - p_j^{d,scaled}\| = s \cdot \|p_i^d - p_j^d\|$$

これにより、以下の問題が発生します：

- **エージェント間の距離が縮小**：衝突のリスク増加
- **デッドロックの発生**：制御バリア関数の制約により動けなくなる

### 1.2 非等方スケーリング（面積保存）の解決策

非等方スケーリングでは、異なる方向に異なるスケーリング係数を適用します：

- **横方向（進行方向に垂直）**：$s_y$ で縮小（狭い道に対応）
- **進行方向**：$s_x = \frac{1}{s_y}$ で拡大（面積保存）

これにより：

- **エージェント間の距離を保持**：安全性確保
- **面積を保存**：自然な動き
- **デッドロック回避**：安定した制御

## 2. 非等方スケーリングの数学的基礎

### 2.1 回転行列

進行方向（リーダーの速度ベクトル）を基準に回転角度を計算します：

```python
velocity_vector = self.leader_velocity
theta = np.arctan2(velocity_vector[1], velocity_vector[0])
```

回転行列：

$$R(\theta) = \begin{pmatrix} \cos\theta & -\sin\theta \\ \sin\theta & \cos\theta \end{pmatrix}$$

逆回転行列：

$$R^{-1}(\theta) = R(-\theta) = \begin{pmatrix} \cos\theta & \sin\theta \\ -\sin\theta & \cos\theta \end{pmatrix}$$

### 2.2 非等方スケーリング行列

```python
sy = self.scaling_factor_y
sx = self.compute_scaling_factor_x(sy)  # sx = 1.0 / max(sy, 0.5)

S = np.array([[sx, 0], [0, sy]])
```

非等方スケーリング行列：

$$S = \begin{pmatrix} s_x & 0 \\ 0 & s_y \end{pmatrix}$$

ここで：
- $s_y = s(d_{min})$：LiDAR検知に基づいて計算（横方向のスケーリング）
- $s_x = \frac{1.0}{\max(s_y, 0.5)}$：面積保存の条件から計算（進行方向のスケーリング）

### 2.3 面積保存の条件

行列式が1になることで面積が保存されます：

$$\det(S) = s_x \cdot s_y = \frac{1.0}{\max(s_y, 0.5)} \cdot s_y = 1.0$$

$s_y \geq 0.5$ の場合：

$$\det(S) = \frac{1.0}{s_y} \cdot s_y = 1.0 \quad \checkmark$$

## 3. 変換の適用手順

### 3.1 実装コード

```python
def compute_scaled_target_position(self, agent_idx):
    # 1. 基準点（重心）
    center = np.mean(self.positions, axis=0)
    
    # 2. 相対位置ベクトル
    relative_pos = self.target_positions[agent_idx] - center
    
    # 3. 回転角 theta の決定（リーダーの速度ベクトルから）
    velocity_vector = self.leader_velocity
    theta = np.arctan2(velocity_vector[1], velocity_vector[0])
    
    # 回転行列 R
    c, s = np.cos(theta), np.sin(theta)
    R = np.array([[c, -s], [s, c]])
    R_inv = np.array([[c, s], [-s, c]])  # 逆回転
    
    # 4. 非等方スケーリング行列 S の決定
    sy = self.scaling_factor_y
    sx = self.compute_scaling_factor_x(sy)
    S = np.array([[sx, 0], [0, sy]])
    
    # 5. 変換の適用
    transformed_pos = R @ S @ R_inv @ relative_pos
    
    # 6. グローバル座標へ
    return transformed_pos + center
```

### 3.2 変換ステップの詳細

**ステップ1：相対位置を計算**

$$\mathbf{r}_i = p_i^d - p_{center}$$

**ステップ2：進行方向をX軸に合わせるように逆回転**

$$\mathbf{r}_i' = R^{-1}(\theta) \cdot \mathbf{r}_i$$

この変換により、進行方向がX軸に、垂直方向がY軸に合わせられます。

**ステップ3：非等方スケーリングを適用**

$$\mathbf{r}_i'' = S \cdot \mathbf{r}_i' = \begin{pmatrix} s_x & 0 \\ 0 & s_y \end{pmatrix} \begin{pmatrix} r_i'x \\ r_i'y \end{pmatrix} = \begin{pmatrix} s_x \cdot r_i'x \\ s_y \cdot r_i'y \end{pmatrix}$$

- X方向（進行方向）：$s_x$ 倍に拡大
- Y方向（横方向）：$s_y$ 倍に縮小

**ステップ4：進行方向に合わせるように回転**

$$\mathbf{r}_i^{scaled} = R(\theta) \cdot \mathbf{r}_i''$$

この変換により、スケーリング後の相対位置を元の進行方向に合わせます。

**ステップ5：グローバル座標に変換**

$$p_i^{d,scaled} = \mathbf{r}_i^{scaled} + p_{center}$$

### 3.3 数値例

**初期条件**：
- 重心：$p_{center} = (0, 0)$
- Agent 0 の目標位置：$p_0^d = (3.0, 0.0)$
- リーダーの速度：$\mathbf{v} = (1.0, 0.0)$（X方向）
- LiDAR検知：$d_{min} = 0.8$ m → $s_y = 0.7$
- 計算：$s_x = \frac{1.0}{0.7} \approx 1.43$

**ステップ1：相対位置**

$$\mathbf{r}_0 = (3.0, 0.0) - (0, 0) = (3.0, 0.0)$$

**ステップ2：逆回転**

$$\theta = \arctan2(0, 1.0) = 0°$$

$$R^{-1}(0°) = \begin{pmatrix} 1 & 0 \\ 0 & 1 \end{pmatrix}$$

$$\mathbf{r}_0' = \begin{pmatrix} 1 & 0 \\ 0 & 1 \end{pmatrix} \begin{pmatrix} 3.0 \\ 0.0 \end{pmatrix} = \begin{pmatrix} 3.0 \\ 0.0 \end{pmatrix}$$

**ステップ3：非等方スケーリング**

$$\mathbf{r}_0'' = \begin{pmatrix} 1.43 & 0 \\ 0 & 0.7 \end{pmatrix} \begin{pmatrix} 3.0 \\ 0.0 \end{pmatrix} = \begin{pmatrix} 4.29 \\ 0.0 \end{pmatrix}$$

**ステップ4：回転**

$$\mathbf{r}_0^{scaled} = \begin{pmatrix} 1 & 0 \\ 0 & 1 \end{pmatrix} \begin{pmatrix} 4.29 \\ 0.0 \end{pmatrix} = \begin{pmatrix} 4.29 \\ 0.0 \end{pmatrix}$$

**ステップ5：グローバル座標**

$$p_0^{d,scaled} = \begin{pmatrix} 4.29 \\ 0.0 \end{pmatrix} + \begin{pmatrix} 0 \\ 0 \end{pmatrix} = \begin{pmatrix} 4.29 \\ 0.0 \end{pmatrix}$$

**結果の比較**：

| スケーリング方法 | 目標位置 |
|-----------------|---------|
| 等方（$s = 0.7$） | $(2.1, 0.0)$ |
| 非等方（$s_x = 1.43, s_y = 0.7$） | $(4.29, 0.0)$ |

非等方スケーリングでは、進行方向に伸びるため、より前に進もうとします。

### 3.4 複数エージェントの場合

**初期状態**：
- Agent 0：$p_0^d = (3.0, 0.0)$
- Agent 1：$p_1^d = (3.0, 0.5)$
- Agent 2：$p_2^d = (3.0, -0.5)$
- 重心：$p_{center} = (3.0, 0.0)$

**相対位置**：
- Agent 0：$\mathbf{r}_0 = (0.0, 0.0)$
- Agent 1：$\mathbf{r}_1 = (0.0, 0.5)$
- Agent 2：$\mathbf{r}_2 = (0.0, -0.5)$

**非等方スケーリング後**（$s_x = 1.43, s_y = 0.7$）：
- Agent 0：$\mathbf{r}_0^{scaled} = (0.0, 0.0)$ → $p_0^{d,scaled} = (3.0, 0.0)$
- Agent 1：$\mathbf{r}_1^{scaled} = (0.0, 0.35)$ → $p_1^{d,scaled} = (3.0, 0.35)$
- Agent 2：$\mathbf{r}_2^{scaled} = (0.0, -0.35)$ → $p_2^{d,scaled} = (3.0, -0.35)$

**エージェント間の距離**：

$$\|p_1^{d,scaled} - p_2^{d,scaled}\| = \|(0, 0.7)\| = 0.7 \text{ m}$$

**結果**：等方スケーリングと同じ距離が保持されます！

## 4. スケーリング係数の計算

### 4.1 横方向のスケーリング係数（$s_y$）

```python
def compute_scaling_factor_y(self, min_obstacle_distance):
    if min_obstacle_distance < self.d_critical:
        return self.s_min
    elif min_obstacle_distance <= self.d_free:
        ratio = (min_obstacle_distance - self.d_critical) / (self.d_free - self.d_critical)
        return self.s_min + (1.0 - self.s_min) * ratio
    else:
        return 1.0
```

$$s_y(d_{min}) = \begin{cases}
s_{min} & \text{if } d_{min} < d_{critical} \\
s_{min} + (1 - s_{min}) \cdot \frac{d_{min} - d_{critical}}{d_{free} - d_{critical}} & \text{if } d_{critical} \leq d_{min} \leq d_{free} \\
1.0 & \text{if } d_{min} > d_{free}
\end{cases}$$

**パラメータ**：
- $s_{min} = 0.7$：最小スケーリング係数（最大30%に縮小）
- $d_{critical} = 1.2$ m：スケーリング開始距離
- $d_{free} = 2.4$ m：スケーリング終了距離

### 4.2 進行方向のスケーリング係数（$s_x$）

```python
def compute_scaling_factor_x(self, sy):
    return 1.0 / max(sy, 0.5)
```

$$s_x = \frac{1.0}{\max(s_y, 0.5)}$$

**意味**：
- $s_y = 0.7$ の場合：$s_x = \frac{1.0}{0.7} \approx 1.43$（43%に拡大）
- $s_y = 1.0$ の場合：$s_x = \frac{1.0}{1.0} = 1.0$（スケーリングなし）

**下限設定**：$\max(s_y, 0.5)$ により、$s_x$ の上限を2.0に制限

### 4.3 面積保存の確認

```python
area_product = history['scaling_factors_x'] * history['scaling_factors_y']
```

面積保存が成立していることを確認：

$$s_x \times s_y = \frac{1.0}{\max(s_y, 0.5)} \times s_y = 1.0$$

## 5. 進行方向の決定

### 5.1 リーダーの速度ベクトルから角度を計算

```python
velocity_vector = self.leader_velocity
theta = np.arctan2(velocity_vector[1], velocity_vector[0])
```

リーダーが $(v_x, v_y)$ の速度で移動している場合：

$$\theta = \arctan2(v_y, v_x)$$

**例**：
- リーダーが $(1.0, 0.0)$ で移動：$\theta = 0°$（X軸方向）
- リーダーが $(1.0, 1.0)$ で移動：$\theta = 45°$（北東方向）
- リーダーが $(0.0, 1.0)$ で移動：$\theta = 90°$（Y軸方向）

### 5.2 リーダーが存在しない場合の代替案

**方法1：エージェント群の平均速度**

```python
velocity_vector = np.mean(self.velocities, axis=0)
theta = np.arctan2(velocity_vector[1], velocity_vector[0])
```

**方法2：エージェント群の進行方向**

```python
direction = np.mean(self.target_positions, axis=0) - np.mean(self.positions, axis=0)
theta = np.arctan2(direction[1], direction[0])
```

**方法3：固定方向**

```python
theta = 0  # X軸方向
```

## 6. 統合制御則

### 6.1 制御入力の計算フロー

```python
def step(self, current_time, dt=0.01, sensing_radius=2.0, use_constraint=True):
    # 1. LiDARで障害物距離を測定
    min_distances = []
    for i in range(self.n_agents):
        distances, angles = self.lidar.measure_distances(
            self.positions[i], self.obstacles
        )
        min_dist = np.min(distances)
        min_distances.append(min_dist)
    
    # 2. スケーリング係数を計算
    self.min_obstacle_distance = np.mean(min_distances)
    self.scaling_factor_y = self.compute_scaling_factor_y(self.min_obstacle_distance)
    self.scaling_factor_x = self.compute_scaling_factor_x(self.scaling_factor_y)
    
    # 3. 各エージェントの制御入力を計算
    for i in range(self.n_agents):
        # 4. スケーリングされた目標状態を計算
        target_pos = self.compute_scaled_target_position(i)
        target_vel = self.compute_scaled_target_velocity(i, current_time, dt)
        target_acc = self.compute_scaled_target_acceleration(i, current_time, dt)
        
        # 5. バックステッピング制御則を計算
        u, e_p, e_v = self.compute_backstepping_control(
            i, target_pos, target_vel, target_acc
        )
        
        # 6. GCBF制約を適用
        if use_constraint:
            neighbors = self.get_neighbors(i, sensing_radius)
            u = self.project_control_to_constraint(u, i, neighbors, alpha=1.0, dt=dt)
        
        # 7. 制御入力を制限
        u_max = 1.5
        u_norm = np.linalg.norm(u)
        if u_norm > u_max:
            u = u * (u_max / u_norm)
        
        u_all[i] = u
    
    # 8. 状態を更新
    self.velocities += u_all * dt
    self.positions += self.velocities * dt
```

### 6.2 制御則の数式

$$u_i = -k_v e_{v,i} - e_{p,i}^{scaled} + a_i^{d,scaled}$$

ここで：
- $e_{p,i}^{scaled} = p_i - p_i^{d,scaled}$：スケーリングされた目標位置との誤差
- $a_i^{d,scaled}$：スケーリングされた目標位置の加速度

## 10. 実装上の注意点

### 10.1 数値安定性

- **スケーリング係数の範囲制限**：$0.5 \leq s_y \leq 1.0$, $1.0 \leq s_x \leq 2.0$
- **距離の下限設定**：$d_{min} \geq \epsilon$（ゼロ除算回避）
- **面積保存の確認**：$|s_x \times s_y - 1.0| < 0.01$

### 10.2 計算効率

- **LiDAR測定の間隔**：全ステップで測定するのではなく、数ステップごとに測定
- **キャッシング**：計算結果を再利用

### 10.3 安全性

- **最小スケーリング係数**：$s_{min} > 0.5$（目標位置が完全に消滅しない）
- **制御バリア関数との組み合わせ**：LiDARスケーリングと衝突回避制約を統合

## 11. まとめ

非等方スケーリング（面積保存）は、以下の利点を提供します：

| 利点 | 説明 |
|------|------|
| **エージェント間の距離保持** | 衝突リスク低減 |
| **面積保存** | 自然な動き |
| **デッドロック回避** | 安定した制御 |
| **進行方向への拡大** | より前に進める |

このアプローチにより、狭い道をスムーズに、かつ安全に通過できるようになります。
